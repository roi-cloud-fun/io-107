# IO-107 SDLC Pipeline -- lab_env_student

> **Generated by labforge untagged on 2026-05-25.** Re-run `labforge/python/generate_setup_artifacts.py --course <CourseDir> --mode student` to refresh.

**Unified student-mode lab environment.** One `terraform apply` provisions everything IO-107 SDLC Pipeline needs: VPC, EKS cluster, ECR, KMS keys, security groups, Aurora subnet group, CodeBuild + CodePipeline service roles, and per-lab artifacts (pipeline + CodeBuild project + S3 artifact bucket + IRSA roles + K8s namespaces + Aurora cluster).

## When to use this mode

- **LTF / smoke tests** -- one apply, one student-id, one set of resources to verify the labs work end-to-end.
- **Instructor demo / dry-run** -- provision a fresh cohort-of-one to walk through every lab yourself.
- **Single-student delivery** -- just hand the whole module to one student.

## When NOT to use this mode

For multi-student delivery (real SYF cohort), use `--mode split` instead. That produces a `training_env/` (one apply per region, shared across cohorts) plus a `student_bootstrap/` (one apply per student per cohort). The split mode is cheaper for shared infra (one EKS cluster, not 25) but more complex to orchestrate.

## Prerequisites (you must do these MANUALLY before `terraform apply`)

1. **Bootstrap Terraform state backend** (one-time per account): create an S3 bucket. Terraform 1.10+ supports native S3 locking via `use_lockfile = true` -- no DynamoDB lock table required. Add a `backend.tf` to this folder pointing at the bucket.

2. **Create a CodeStar source connection** in the AWS Console:
   - Console → CodePipeline → Settings → Connections → Create
   - Provider: GitHub (or whichever your labs target)
   - Click **Update pending connection** and complete the OAuth handshake
   - Copy the resulting ARN -- pass it as `github_codestar_connection_arn`

3. **Have your `student_id` ready** (lowercase, 1-16 chars). For LTF, conventionally `ltf-${TIMESTAMP}` or `ltf-smoke`.

## Apply

```bash
cp terraform.tfvars.example terraform.tfvars
# edit terraform.tfvars: student_id, github_codestar_connection_arn

terraform init
terraform plan -out=tfplan
terraform apply tfplan
```

EKS provisioning takes **~15 minutes** the first time. Plan accordingly.

## Outputs

After `apply` succeeds, capture the outputs -- LTF and the lab guides reference them:

```bash
terraform output -json > outputs.json
```

Important outputs:

- `eks_cluster_name` -- referenced by every lab buildspec
- `kubeconfig_command` -- copy-paste to put the cluster in your local kubeconfig
- `ecr_repos` -- map of app name → registry URL
- `lab{N}_pipeline_name` -- per-lab CodePipeline (LTF watches these)
- `lab{N}_artifact_bucket` -- per-lab pipeline artifact bucket
- `lab{N}_namespace` -- per-lab K8s namespace (where IRSA workloads land)
- `lab{N}_aurora_cluster_id` -- Aurora cluster (Lab 4 Blue/Green target)


## Tear-down

```bash
terraform destroy
```

Everything is tagged `StudentId=<your-id>` for audit, but `destroy` removes the entire stack -- VPC, EKS, ECR, etc. The CodeStar connection is NOT torn down (it's a manual resource you pre-created).
